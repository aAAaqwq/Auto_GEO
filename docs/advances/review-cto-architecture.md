# CTO 技术架构审查报告

_审查日期: 2026-05-06 | 审查人: CTO_

---

## 🔴 P1 风险（需尽快处理）

### 1. SQLite 并发瓶颈
- 单用户/桌面端场景尚可，多用户并发写入时 WAL 模式仍会锁竞争
- 生产环境约 75% 概率出现锁竞争问题
- **建议**: 短期优化 WAL 配置；中期评估 PostgreSQL 迁移路径

### 2. 加密密钥硬编码
- `crypto.py` 中密钥管理方式存在安全隐患
- 密钥硬编码或弱派生，泄露即全盘暴露
- **建议**: 使用环境变量 + KMS 密钥管理服务

### 3. Playwright 单实例瓶颈
- 浏览器自动化使用单实例，无法并行发布
- 多任务排队，发布效率低
- **建议**: 实例池化管理，支持多浏览器上下文并行

## 🟡 P2 问题

### 4. WebSocket 无鉴权
- WebSocket 连接未做身份验证，存在安全风险
- **建议**: 连接时校验 token

### 5. Docker 镜像过大
- 包含 Playwright 浏览器二进制，镜像体积大
- **建议**: 多阶段构建，分离运行时和构建依赖

### 6. CI 无测试
- GitHub Actions 只有 lint 和 build，无测试步骤
- **建议**: 加入 pytest + playwright test

### 7. CORS 硬编码
- 跨域配置写死，不利于多环境部署
- **建议**: 从环境变量读取允许的 origins

## ✅ 做得好的
- Docker 部署基础完善
- 安全基础配置（gitleaks、.env.example）
- 架构分层清晰（api / services / database）
